# Precision Automation Labs — Antigravity Redesign Brief

## Purpose

This file is the source-of-truth brief for a **full visual redesign** of the Precision Automation Labs website.

The redesign should **change the look, layout, motion system, art direction, and overall experience from top to bottom** while **preserving the existing PAL copy and content structure unless explicitly noted otherwise**.

This is **not** a light facelift.
This is a **premium, cinematic, pinned-section-first redesign** intended to make the site feel more immersive, more advanced, and more distinctive.

---

## Primary Goal

Redesign the site so it feels:

- premium
- cinematic
- technical
- precise
- editorial but futuristic
- highly intentional
- visually unified from first load through footer

The entire site should feel like a **designed system**, not a collection of standard web sections.

Pinned sections should be a **core design language across the site**, not just a one-off effect.

---

## Non-Negotiables

### 1) Keep the copy
Keep the existing homepage copy, section labels, messaging, and PAL voice.

Do **not** rewrite the site into generic AI-agency language.
Do **not** replace PAL’s QA automation framing with “AI systems” messaging.

The copy can be:
- reorganized
- redistributed across sections
- broken into stronger visual groupings
- emphasized differently

But the **voice and message must remain PAL**.

### 2) Preserve the project stack and repo direction
This project is already being rebuilt as:

- Next.js app router
- TypeScript / `.tsx`
- modular section components
- modular layout components
- shared content/data file

Do not collapse the project back into one giant page file.

### 3) Pinned sections should drive the experience
The new design should use pinned sections as a **primary interaction pattern** from the hero downward.

The site should feel like the user is moving through a carefully staged sequence of scenes.

### 4) Premium over gimmicky
The redesign should feel elevated and intentional.

Avoid:
- cheesy sci-fi
- neon overload
- generic SaaS blocks
- cluttered dashboards
- stock-template agency design
- juvenile “AI” visuals

The site should feel more like:
- a precision lab
- a technical manifesto
- an advanced editorial product site
- a controlled systems environment

## Experience Model

## Overall structure
The site should feel like a **sequence of pinned scenes**.

Think in terms of:

1. Arrival
2. Framing the philosophy
3. Breaking down the operating model
4. Showing tools and systems
5. Showing publishing / updates / proof of thinking
6. Closing with a high-confidence contact / CTA scene

The user should feel guided through a narrative.

## Scroll behavior
- Use pinned sections generously.
- Use layered entrances/exits inside pinned scenes.
- Use transitions that feel like handoffs between systems.
- Maintain strong pacing.
- Avoid random floating animations with no structural reason.

## Motion principles
Motion should feel:
- deliberate
- inertial
- elegant
- engineered
- cinematic

Avoid:
- bouncy toy-like motion
- overuse of blur for no reason
- chaotic parallax
- too many things moving at once

### Preferred motion patterns
- pinned scenes with enter / hold / exit phases
- layered staggered reveals
- directional motion based on layout rhythm
- subtle depth shifts
- scene-to-scene continuity
- line growth / node activation / data-flow style transitions
- understated glow and highlight movement

## Technical / Implementation Guidance

- TypeScript

- modular layout components

## Motion / implementation guidance

## Maintainability requirement
Even though the redesign should be dramatic, the result should still be maintainable by a developer who wants:
- full-file replacements when iterating
- clear section boundaries
- understandable motion ownership
